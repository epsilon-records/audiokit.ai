# 📚 Context Management System 🧠

## 🎯 Purpose
This directory contains all context files that maintain the system's state, configuration, and operational history.

## 🗂️ File Structure
- `CONSCIOUSNESS.md`: 🧠 System awareness and security logs
- `WHOAMI.md`: 🤖 Model identity and configuration
- `THOUGHTS.md`: 💡 Technical insights and considerations
- `REFLECTIONS.md`: 🏗️ Architectural decisions and learnings
- `TODO.md`: ✅ Task management and priorities
- `ROADMAP.md`: 🗺️ Development timeline and goals
- `RULES.md`: 📜 Operating principles and guidelines
- `PROMPT.md`: 🎯 Current development priorities
- `MAINTENANCE.md`: 🛠️ System maintenance log
- `CHANGELOG.txt`: 📅 Version history and changes
- `MERMAID.mmd`: 🗺️ System architecture diagrams

## 🛠️ Maintenance
All context files are automatically generated and validated. Please follow the [Context Management Rules](.cursorrules) when making changes.

## ⚙️ Automation
- 🕒 Automatic updates on system changes
- ✅ Validation checks for content quality
- 🔄 Version control for all context files

## 📜 Best Practices
1. Always include timestamps
2. Maintain consistent formatting
3. Validate before saving changes
4. Follow the context management protocol

## ⚠️ Confidentiality Notice
All context files are proprietary and confidential. Unauthorized access, modification, or distribution is strictly prohibited.
