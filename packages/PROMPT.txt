AudioKit Development Guidelines
=============================

Development Priorities
--------------------
1. Implement client and server features in parallel to maintain compatibility
2. Complete one TODO item from each package before moving to next items
3. Update documentation as features are implemented
4. Add tests only when features are stable
5. Focus on core functionality before optimizations

Code Quality Standards
--------------------
1. Follow PEP 8 style guide
2. Use type hints consistently
3. Write comprehensive docstrings
4. Keep functions focused and small
5. Use meaningful variable names
6. Handle errors gracefully
7. Write tests for all functionality
8. Use async/await for I/O operations
9. Follow SOLID principles
10. Use dependency injection

Documentation Requirements
------------------------
1. Module/Class/Function Documentation:
   - Purpose and description
   - Parameters and return types
   - Usage examples
   - Error conditions
   - Performance considerations

2. Architecture Documentation:
   - Component diagrams
   - Data flow descriptions
   - API specifications
   - Deployment guides

Testing Requirements
------------------
1. Unit Tests:
   - 100% coverage for core functionality
   - Property-based testing where appropriate
   - Mock external dependencies

2. Integration Tests:
   - API endpoint testing
   - End-to-end workflows
   - Performance benchmarks

3. Security Tests:
   - Input validation
   - Authentication/Authorization
   - Rate limiting
   - Dependency scanning

Code Review Checklist
-------------------
1. Code Quality:
   - Follows style guide
   - Uses type hints
   - Has docstrings
   - Handles errors
   - Is testable

2. Testing:
   - Has unit tests
   - Has integration tests
   - Maintains coverage
   - Tests edge cases

3. Security:
   - Validates input
   - Sanitizes output
   - Uses secure defaults
   - Follows best practices

4. Performance:
   - Uses async where appropriate
   - Optimizes resource usage
   - Considers scalability
   - Has benchmarks

Development Process
-----------------
1. Create feature branch
2. Write tests first
3. Implement functionality
4. Format and lint code
5. Run all tests
6. Update documentation
7. Submit pull request
8. Address review feedback
9. Merge to main branch
